## 1. What ORI Is

ORI (Operational Reasoning Intelligence) is a governed, modular intelligence framework.

It is NOT:
- An autonomous AI
- A trading bot
- A system-level agent

It IS:
- A controlled orchestration system
- A secure execution framework
- A modular agent architecture
- A memory-augmented reasoning system

---

## 2. Core Philosophy

ORI is built on five non-negotiable principles:

1. Modularity
2. Explicit approval
3. Reduced attack surface
4. Separation of orchestration and execution
5. Governed intelligence

---

## 3. Two-Machine Architecture

### OriCore
- Orchestration layer
- Approval gate
- Agent coordination
- Job submission
- User interface

### OriCN
- Execution layer
- Strict ACTION_HANDLERS whitelist
- No user interface
- No external network calls
- File confinement to project root

---

## 4. Governance Model

### System-Level Governance
- Linux permissions
- SSH keys
- Network segmentation
- No exposed services

### Software-Level Governance
- Explicit action routing
- Approval gate
- Structured job lifecycle
- Immutable audit logs
- No arbitrary code execution

---

## 5. Security Constraints

ORI may:
- Analyze
- Summarize
- Recommend

ORI may NOT:
- Execute financial transactions
- Access broker APIs
- Make autonomous decisions
- Modify system state without approval
